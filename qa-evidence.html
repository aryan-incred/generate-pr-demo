mm-qa-gatekeeper — How It Ensures Correctness & Efficiency
Invoked by /mm-ship · Phases 5, 6, 7

mm-qa-gatekeeper is a specialised subagent in InCred's agentic SDLC pipeline. It runs autonomously on each environment promotion, producing structured evidence before any human-approval gate fires. Haiku modelPhase 5–7

How it ensures correctness
  • Reads PLAN.md Section 3 for exact p95 targets and test scenarios — never invents thresholds
  • Runs the actual test command from PLAN.md IMPLEMENTATION INFO (npm run test) — not a simulation
  • Validates each integration test against live dynamo-svc and document-service in the target environment
  • Checks regression scenarios explicitly — runs the old endpoints and compares response shapes byte-for-byte
  • Never silently approves — any failure surfaces root cause and blocks promotion until fixed
  • Signs BUILD-EVIDENCE + QA-EVIDENCE as a machine-auditable paper trail
How it ensures efficiency
  • Uses Haiku (fast, low-cost) for test execution — Opus/Sonnet reserved for design phases
  • Reads PLAN.md Section 3 directly — no re-derivation of requirements, zero back-and-forth
  • Runs unit, integration, regression, and p95 in a single unattended pass — no human hand-off between sub-gates
  • Skips lint/format checks if not in package.json scripts — no false failures on unsupported tools
  • Partial results surface immediately — developer can fix and re-run just the failing suite
  • Context passed directly (no re-reads of source code) — minimal token cost per gate run
Gate execution flow
  1. Read PLAN.md Section 3
    Extract p95 targets, unit/integration/regression scenarios, test command, services in scope
  2. Deploy branch to environment
    Pull from base branch, verify deployment health check responds
  3. Run unit tests
    Execute npm run test (or equivalent); require 100% of story tests to pass
  4. Run integration tests
    Hit live endpoints with golden data from PLAN.md Section 3 AI Acceleration Strategy
  5. Run regression suite
    Verify all endpoints listed in PLAN.md Section 3 Regression Scenarios still behave identically
  6. Measure p95 latency
    50 samples against each integration scenario; compare to PLAN.md targets; hard-fail if exceeded
  7. Write QA-EVIDENCE.md
    Structured report with all results — committed to monorepo before human sign-off gate fires
